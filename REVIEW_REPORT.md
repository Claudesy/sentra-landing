## Remaining Recommendations

### Medium Priority
- **Automated testing:** Add Vitest for unit tests + Playwright for E2E. Target ≥80% coverage threshold in CI.
- **OpenTelemetry:** Instrument with `@opentelemetry/sdk-node` when the site grows API routes.
- **Server Components:** Evaluate converting static sections (Navbar, Footer, Clients) from `"use client"` to RSC to reduce JS bundle.
- **CD secrets:** Configure `RAILWAY_TOKEN` (or equivalent) in GitHub Environment `production` to activate `cd.yml`.

### Low Priority
- **Bundle analyser:** Add `ANALYZE=true npm run build` capability via `@next/bundle-analyzer`.
- **Structured logging:** Add `pino` when API routes are introduced.
- **ADR backlog:** Document the decision to use Framer Motion over CSS animations (ADR-002).
